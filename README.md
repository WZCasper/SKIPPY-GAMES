# SkippyGames

Магазин игр (Steam / PlayStation / Xbox / Nintendo Switch) и внутриигровой
валюты на GitHub Pages.

## Структура

```
index.html                         — главная (хиты, каталог, фильтры, валюта, о нас)
game.html                          — отдельная страница карточки игры (реальная навигация, не попап)
common.js                          — общий код: VK-чат, wishlist, toast, утилиты
home.js                            — логика главной страницы
game.js                            — логика страницы карточки игры
data/index.json                    — лёгкий индекс каталога (для списка/поиска/фильтров)
data/games/{id}.json               — полные данные одной игры (описание, скриншоты, трейлер, DLC)
data/currency.json                 — агрегированные DLC/донат-паки всех игр, для раздела "Валюта"
backend/parser.py                  — сборщик данных Steam -> data/*
backend/custom_ids.txt             — appid, добавляемые вручную
backend/skipped_ids.json           — appid, не являющиеся играми (создаётся автоматически)
backend/requirements.txt           — зависимости Python
.github/workflows/update_games.yml — автозапуск парсера каждые 6 часов
```

## Важное честное замечание про дизайн

По вашей просьбе я открывал `wzcasper.github.io/SkippyGame` — но мой
инструмент для чтения страниц возвращает только извлечённый текст, а не
исходные CSS/JS файлы. Поэтому цвета, шрифты и пиксельная вёрстка того
сайта не были доступны буквально. Я воссоздал ту же структуру разделов и
весь функционал (шапка с навигацией по хитам/каталогу/валюте/о нас,
wishlist, hero со статистикой, горящие предложения, каталог с фильтрами и
сортировкой, кнопка загрузки полного каталога со счётчиком, раздел валюты
с категориями, блок "о магазине", футер, чат) в уже согласованной ранее
тёмно-жёлтой геймерской стилистике. Если у вас есть исходный код того
сайта (например, вы можете скачать его через "Add file → Upload files" на
GitHub или прислать мне HTML-файл) — пришлите, и я подгоню вёрстку 1:1.

## Что исправлено в этом обновлении

### 1. Карточки игр — минимальное затемнение

Раньше градиент почти полностью перекрывал обложку
(`bg-gradient-to-t from-neutral-800 to-transparent` на весь `inset-0`).
Теперь класс `.cover-shade` — узкая полоса высотой 30% от низа карточки с
мягким градиентом, нужная только для читаемости иконок платформ поверх
обложки.

### 2. Реальная навигация вместо попапа

Раньше клик по карточке переключал `<div>` внутри одной страницы (SPA).
Теперь `game.html` — самостоятельная HTML-страница. Карточки — обычные
`<a href="game.html?id=...">`, то есть настоящий переход браузера, с
собственным URL, кнопкой "назад" браузера и возможностью открыть в новой
вкладке (Ctrl+клик) — как и просили: "standard href".

### 3. Наценка — строго и прозрачно +500 ₽

В `backend/parser.py` расчёт цены переписан так, чтобы формула была
самоочевидной и защищённой от регрессий:

```python
base_price_rub = round(price_uah * uah_to_rub)  # цена из Steam в рублях
markup_rub = 500                                 # фиксированная наценка
price_rub = base_price_rub + markup_rub          # итоговая цена
assert price_rub == base_price_rub + markup_rub  # гарантия на будущее
```

Оба поля (`base_price_rub`, `markup_rub`) сохраняются в
`data/games/{id}.json` рядом с итоговой ценой — на странице игры теперь
прямо под ценой показывается расшифровка: *"Цена в Steam: 2300 ₽ +
сервисный сбор 500 ₽ = 2800 ₽"*, чтобы наценка была видна и проверяема, а
не пряталась внутри одного числа. То же самое сделано для DLC/донат-паков.
Бесплатные (free-to-play) игры остаются бесплатными — наценка на них не
начисляется (0 + 0 = 0).

### 4. VK-чат — более надёжная загрузка

Так как настройки на стороне сообщества ВК вы уже проверили, я сфокусировался
на всём, что можно исправить в коде:

- Скрипт VK теперь **прогревается в фоне** через 1.5 сек после загрузки
  страницы (а не только по клику на чат) — это убирает возможную "гонку"
  между кликом пользователя и временем загрузки `openapi.js`.
- Убрана жёсткая версия скрипта `?169` в URL — используется канонический
  `https://vk.com/js/api/openapi.js`, который VK обслуживает как актуальную
  версию (старый закреплённый номер версии со временем может устареть).
- Добавлена проверка по таймауту: если через 6 секунд после вызова
  `VK.Widgets.CommunityMessages` внутри контейнера так и не появился
  `<iframe>` (который создаёт сам виджет VK) — код это обнаруживает и
  показывает понятную заглушку с прямыми ссылками на VK/Telegram и кнопкой
  "Попробовать снова", вместо пустого окна.
- В консоль браузера (F12 → Console) при сбое выводится точная причина:
  "скрипт не загрузился", "виджет не ответил вовремя" и т.д. — это поможет
  быстро продиагностировать, если проблема всё же возникнет.

**Важно понимать:** если после всего этого виджет всё ещё не грузится —
это почти наверняка блокировщик рекламы в браузере (расширения вроде
uBlock Origin по умолчанию блокируют виджеты `vk.com`, так как относят их
к категории "трекеров соцсетей", вместе с виджетами Facebook/Twitter).
Проверьте в режиме инкогнито без расширений — это позволит отличить
проблему кода от блокировки в конкретном браузере.

### 5. Остальные функции с прошлого обновления сохранены

Трейлер (Steam → YouTube Data API → честная ссылка на поиск), лайтбокс
скриншотов, полные описания без обрезки, DLC/допродажи, футер с указанием
владельца и Telegram, высокое разрешение обложек/скриншотов — всё это
перенесено без изmenений в новую структуру страниц.

## Новые разделы на главной странице

- **🔥 Хиты** — горящие предложения: игры с реальной скидкой Steam
  (`discount_percent > 0`), а не случайные/выдуманные позиции.
- **🎮 Каталог** — фильтры по платформе, жанрам (44 штуки, выпадающий
  список), максимальной цене, чекбокс "только бесплатные", сортировка
  (по умолчанию / цена ↑ / цена ↓ / А-Я), пагинация.
- **📦 Загрузить полный каталог Steam** — рендерит весь отфильтрованный
  список игр порциями по 200 карточек с живым счётчиком прогресса. Важно
  честно: сами данные (`data/index.json`) уже загружены целиком одним
  запросом при заходе на сайт — эта кнопка не скачивает что-то новое по
  сети, а порционно отрисовывает уже полученные данные в DOM, чтобы браузер
  не завис при рендере тысяч карточек разом. Это настоящая оптимизация
  производительности, а не имитация процесса.
- **💎 Внутриигровая валюта** — все DLC/донат-паки из каталога,
  агрегированные в `data/currency.json`, с фильтром по категориям (Шутеры /
  MOBA и стратегии / RPG и MMO).
- **👤 О магазине** — карточка владельца (Андрей «ShadowSkip») со ссылками
  на YouTube/Twitch/VK.
- **Wishlist** — список избранного полностью на клиенте (`localStorage`),
  сердечко на карточках и в шапке, выдвижная панель со списком.

## Архитектура данных (без изменений с прошлого раза)

GitHub Pages — статический хостинг без сервера и без настоящей базы данных.
`data/index.json` — единый лёгкий индекс всех игр для каталога/поиска/
фильтров. `data/games/{id}.json` — по одному файлу с "тяжёлыми" данными на
игру, подгружается лениво только при открытии её страницы. Инкрементальный
сбор данных (600 новых игр за прогон, до 10 000+ игр за 1–2 недели) —
подробности в предыдущей версии этого README, логика не менялась.

## Настройка

### Числовой ID группы ВКонтакте

В `common.js`:

```js
var VK_GROUP_NUMERIC_ID = 195484236;
```

### YouTube Data API (фолбэк трейлеров)

Google Cloud Console → включить "YouTube Data API v3" → создать ключ →
добавить как секрет `YOUTUBE_API_KEY` в Settings → Secrets and variables →
Actions репозитория. Без ключа — фолбэк отключён, сайт честно покажет
ссылку "Искать на YouTube" вместо видео.

### Деплой на GitHub Pages

1. Загрузите все файлы в репозиторий (ветка `main`).
2. Settings → Pages → Source → ветка `main`, папка `/ (root)`.
3. Settings → Actions → General → Workflow permissions → "Read and write
   permissions".
4. Actions → "Update game catalog" → Run workflow (первый запуск вручную).
5. Откройте `https://<логин>.github.io/<репозиторий>/`.

## Локальный запуск парсера

```bash
cd backend
pip install -r requirements.txt
export YOUTUBE_API_KEY=ваш_ключ   # необязательно
python parser.py
```
